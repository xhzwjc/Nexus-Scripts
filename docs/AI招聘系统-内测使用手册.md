## 1. 系统访问与登录

### 访问地址

```
http://<服务器地址>:3000
```

> 内网部署请联系管理员获取具体地址。

### 登录步骤

1. 打开浏览器访问上述地址
2. 在 **Key** 输入框中输入你的访问密钥
3. 点击 **验证密钥** 按钮
4. 验证通过后自动进入系统首页

> **注意**：密钥仅显示一次，验证通过后请妥善保存。如遗忘请联系管理员重置。

### 首页概览

登录成功后进入首页，可以看到：
- 系统公告与快捷入口
- 侧边栏导航菜单
- 当前用户信息（右上角）

## 2. 权限中心使用指南

> 入口：侧边栏 → **Access Control**（盾牌图标）

权限中心包含 6 个标签页：**Overview、Organizations、Users、Roles、Resources、Audit**。

### 2.1 添加组织架构

> 入口：权限中心 → **Organizations** 标签

组织架构采用树形结构，支持：集团 → 子集团 → 公司 → 部门 四级结构。

#### 创建组织步骤

1. 点击右上角 **创建组织 / Create Organization** 按钮
2. 填写组织信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 组织代码 | 唯一标识，小写字母/数字/下划线/连字符 | `beijing-tech` |
| 组织名称 | 显示名称 | `北京科技公司` |
| 组织类型 | 集团/子集团/公司/部门 | `company` |
| 上级组织 | 父级组织（留空则为根级） | `group-hq` |
| 排序号 | 数字越小越靠前 | `10` |
| 状态 | 是否启用 | 勾选启用 |

3. 点击 **保存** 完成创建

#### 注意事项

- 组织代码创建后不可修改
- 删除组织前需确保无用户归属该组织
- 下级组织的可见性受上级组织的数据范围控制

---

### 2.2 创建角色

> 入口：权限中心 → **Roles** 标签

角色定义了一组权限的集合，用户通过角色获得相应权限。

#### 创建角色步骤

1. 点击 **创建角色 / Create Role** 按钮
2. 填写基本信息：

| 字段 | 说明 |
|------|------|
| 角色代码 | 唯一标识（小写） |
| 角色名称 | 显示名称 |
| 描述 | 角色用途说明 |

3. 选择权限（按分类勾选）：

| 权限分类 | 权限说明 |
|----------|----------|
| **AI 招聘** | 招聘相关全部权限 |
| **业务工具** | 结算、佣金等业务工具 |
| **资源管理** | 团队资源、AI资源 |
| **平台运维** | 系统监控、健康检测 |

4. 点击 **保存** 完成创建

#### 推荐角色配置（招聘专员）

为普通招聘人员推荐创建 **招聘专员** 角色，赋予以下权限：

| 权限代码 | 权限名称 |
|----------|----------|
| `recruitment-dashboard-view` | 查看招聘仪表盘 |
| `recruitment-candidate-manage` | 候选人管理 |
| `recruitment-process-execute` | 执行初筛、生成面试题 |
| `recruitment-log-view` | 查看操作日志 |
| `recruitment-skill-view` | 查看 Skills |
| `recruitment-skill-bind` | 绑定 Skills 到岗位 |
| `recruitment-mail-view` | 查看邮件配置 |
| `recruitment-mail-send` | 发送邮件 |

### 2.3 创建用户

> 入口：权限中心 → **Users** 标签

#### 创建用户步骤

1. 点击 **创建用户 / Create User** 按钮
2. 填写用户信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 用户代码 | 唯一标识 | `zhangsan` |
| 显示名称 | 界面显示名称 | `张三` |
| 访问密钥 | 登录密码（可自动生成） | 自动生成 |
| 主组织 | 用户归属组织 | `beijing-tech` |
| 数据范围 | 可查看的数据范围 | `SUB_ORG` |
| 角色 | 分配的角色 | 招聘专员 |

#### 数据范围说明

| 选项 | 说明 |
|------|------|
| `SELF` | 仅查看自己创建的数据 |
| `DEPARTMENT` | 本部门及子部门数据 |
| `SUB_ORG` | 主组织及下级组织数据 |
| `COMPANY` | 整个公司数据 |
| `GROUP` | 集团全部数据 |

3. 点击 **保存**
4. **重要**：系统会显示访问密钥，**请立即复制保存**，关闭后无法再次查看

#### 密钥管理

- **重置密钥**：编辑用户 → 输入新密钥或留空自动生成
- 密钥重置后原密钥立即失效

## 3. AI 招聘全流程

> 入口：侧边栏 → **AI 招聘 / AI Recruiting**

### 3.1 选择公司/组织

进入招聘页面后，需先选择目标公司/组织。

1. 点击页面顶部的 **组织选择器**
2. 在弹出的树形结构中选择目标公司或部门
3. 选择后，页面会自动筛选该组织下的所有数据

> **注意**：若无组织选择器，说明当前账号只有一个可用组织，系统自动限定。

---

### 3.2 配置 Skills

> 入口：招聘页面 → 导航栏 **设置** → **Skills**

Skill（技能）是 AI 执行任务的 Prompt 模板。初筛和面试题生成都需要绑定 Skill。

#### 创建 Skill 步骤

1. 点击 **+ 新增 Skill** 按钮
2. 填写 Skill 信息：

| 字段 | 说明 |
|------|------|
| 名称 | Skill 名称 |
| 描述 | 简要说明用途（选填） |
| 内容 | **核心 Prompt 模板**，定义 AI 如何执行任务 |
| 标签 | 分类标签，用英文逗号分隔 |
| 排序号 | 数字越小越靠前 |
| 启用状态 | 是否启用 |

3. 点击 **保存**

#### Skill 类型说明

| Skill 类型 | 用途 | 必填说明 |
|------------|------|----------|
| JD Skill | 生成岗位描述（JD） | 选填 |
| **初筛 Skill** | 简历初筛评分 | 启用自动初筛时必填 |
| 面试题 Skill | 生成面试问题 | 选填 |

#### Skill 内容编写建议

初筛 Skill 示例框架：

```markdown
# 初筛 Skill 示例

## 评估维度
1. 硬性要求匹配度（学历、工作年限、技术栈）
2. 软性能力（沟通、学习、协作）
3. 加分项（项目经验、证书）

## 输出格式
请按以下 JSON 格式输出：
{
  "overall_score": 85,
  "dimensions": [...],
  "summary": "候选人优势与不足",
  "recommendation": "建议面试重点"
}
```

### 3.3 配置大模型

> 入口：招聘页面 → 导航栏 **设置** → **模型**

大模型配置决定了 AI 执行任务时调用的模型。

#### 创建模型配置步骤

1. 点击 **+ 新增模型** 按钮
2. 填写模型信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 配置名称 | 便于识别的名称 | `GPT-4o-初筛` |
| 任务类型 | 应用场景 | `screening_flow` |
| 提供商 | 模型服务商 | `openai` |
| 模型名称 | 具体模型 ID | `gpt-4o` |
| API 密钥环境变量 | 密钥存储方式 | `OPENAI_API_KEY` |
| 优先级 | 数字越小优先级越高 | `1` |
| 启用状态 | 是否启用 | 勾选启用 |

#### 常用任务类型

| 任务类型 | 说明 |
|----------|------|
| `screening_flow` | 简历初筛 |
| `jd_generation` | JD 生成 |
| `interview_question_generation` | 面试题生成 |
| `chat_orchestrator` | AI 助手对话 |

#### 设置当前使用模型

在模型列表中，点击目标模型的 **设为当前使用** 按钮，该任务类型将优先使用此模型。

### 3.4 配置邮件中心

> 入口：招聘页面 → 导航栏 **设置** → **邮件**

#### 第一步：配置发件箱

1. 点击 **+ 新增发件箱** 按钮
2. 填写发件邮箱信息：

| 字段 | 说明 |
|------|------|
| 配置名称 | 便于识别的名称 |
| 发件人名称 | 邮件显示的发件人 |
| 发件邮箱 | SMTP 邮箱地址 |
| SMTP 服务器 | 邮箱 SMTP 地址 |
| SMTP 端口 | 通常 465（SSL）或 587（TLS） |
| 用户名 | SMTP 登录用户名 |
| 密码 | SMTP 登录密码 |
| 使用 SSL/TLS | 加密方式 |

> **常用邮箱预设**：
> - 163 邮箱：选择 `163 默认`
> - QQ 邮箱：手动配置 `smtp.qq.com:465, SSL`
> - Outlook：选择 `Outlook 默认`

3. 点击 **保存**

#### 第二步：添加收件人

1. 点击 **+ 新增收件人** 按钮
2. 填写收件人信息：

| 字段 | 说明 |
|------|------|
| 姓名 | 收件人姓名 |
| 邮箱 | 收件人邮箱地址 |
| 部门 | 所在部门（选填） |
| 职位 | 职位名称（选填） |
| 标签 | 分类标签 |
| 启用状态 | 是否启用 |

3. 点击 **保存**

#### 第三步：配置自动推送（可选）

1. 在 **全局自动推送设置** 中勾选 **启用系统级自动推送能力**
2. 选择默认收件人
3. 点击 **保存默认配置**

> **提示**：自动推送功能需要在岗位配置中启用，且仅当候选人状态允许时触发。

### 3.5 创建招聘岗位

> 入口：招聘页面 → **岗位** 标签页

#### 创建岗位步骤

1. 点击 **+ 新建岗位** 按钮
2. 填写岗位基本信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 所属公司 | 归属组织 | 北京科技公司 |
| 岗位名称 | **必填** | `高级前端工程师` |
| 部门 | 所属部门 | 前端技术部 |
| 地点 | 工作地点 | 北京 |
| 用工类型 | 全职/兼职等 | 全职 |
| 薪资范围 | 薪酬区间 | `25K-40K` |
| 招聘人数 | **必填** | `2` |
| 岗位状态 | 进行中/暂停等 | `recruiting` |

3. 填写岗位描述（可选）：

| 字段 | 说明 |
|------|------|
| 关键要求 | 必达条件 |
| 加分项 | 加分条件 |

4. **配置初筛设置**（关键步骤）：

| 设置项 | 说明 | 注意事项 |
|--------|------|----------|
| 上传简历后自动进入初筛 | 上传简历自动触发初筛 | **必须先绑定初筛 Skill** |
| 初筛通过后自动推进状态 | 自动更新候选人状态 | 建议开启 |
| 初筛完成后自动推送邮件 | 自动发送结果邮件 | 需配置邮件中心 |

5. **绑定 Skills**（关键步骤）：

| Skill 类型 | 说明 | 必填 |
|------------|------|------|
| JD Skill | 生成岗位描述 | 选填 |
| **初筛 Skill** | 简历初筛评分 | 启用自动初筛时必填 |
| 面试题 Skill | 生成面试问题 | 选填 |

> **重要**：如需启用"上传简历后自动进入初筛"，**必须先在下方绑定至少一个初筛 Skill**，否则无法勾选。

6. 点击 **保存** 完成创建

### 3.6 上传简历

> 入口：招聘页面 → **候选人** 标签页

#### 上传步骤

1. 点击 **上传简历** 按钮
2. 在弹出的上传对话框中：

| 设置项 | 说明 |
|--------|------|
| 关联岗位 | 选择简历对应的岗位（可暂不关联） |
| 目标组织 | 未关联岗位时需选择 |

3. 点击 **选择文件**，支持格式：`PDF、DOC、DOCX、TXT`
4. 可一次选择多个文件
5. 点击 **开始上传**

#### 上传结果说明

上传完成后显示结果统计：

| 状态 | 说明 |
|------|------|
| 自动初筛已入队 | 简历已关联岗位且开启了自动初筛 |
| 已跳过进行中任务 | 该候选人已有进行中的初筛任务 |
| 失败 | 上传或初筛失败（会显示原因） |

> **提示**：如上传失败显示"未配置初筛 Skill"，请先到岗位配置中绑定初筛 Skill。

### 3.7 发起初筛

> 入口：招聘页面 → **候选人** 标签页

#### 单个候选人初筛

1. 在候选人列表中勾选目标候选人
2. 点击 **开始初筛** 按钮
3. 系统自动使用岗位绑定的 Skill 执行初筛
4. 可在候选人详情中查看初筛进度和结果

#### 批量初筛

1. 勾选多个候选人
2. 点击 **批量初筛** 按钮
3. 系统自动为每个候选人创建初筛任务

#### 初筛结果查看

初筛完成后，在候选人详情中查看：

| 内容 | 说明 |
|------|------|
| AI 评分 | 综合评分（0-100） |
| 维度评分 | 各评估维度得分 |
| 评语摘要 | AI 给出的评估意见 |
| 建议面试重点 | 后续面试关注点 |

---

### 3.8 生成面试题

> 入口：招聘页面 → **候选人** 标签页 → 打开候选人详情

#### 生成步骤

1. 打开候选人详情页面
2. 在面试题区域填写：

| 字段 | 说明 |
|------|------|
| 面试轮次 | 如"初试"、"复试" |
| 自定义要求 | 补充要求（选填） |

3. 选择面试题 Skill（系统自动使用岗位绑定的 Skill）
4. 点击 **生成面试题** 按钮
5. 等待 AI 生成完成

#### 查看与编辑

生成的面试题以 Markdown 格式展示，支持：
- 直接复制
- 二次编辑修改
- 发送给候选人

## 4. 常见问题

### Q1: 登录失败，密钥无效？

- 检查密钥是否输入正确
- 密钥区分大小写
- 联系管理员确认密钥状态

### Q2: 无法发起初筛？

可能原因：
1. **候选人未绑定岗位** → 先为候选人分配岗位
2. **岗位未绑定初筛 Skill** → 编辑岗位，添加初筛 Skill
3. **候选人无简历** → 先上传简历

### Q3: 无法生成面试题？

可能原因：
1. **候选人未绑定岗位** → 先为候选人分配岗位
2. **岗位未绑定面试题 Skill** → 编辑岗位，添加面试题 Skill

### Q4: 邮件发送失败？

检查项：
1. 发件箱 SMTP 配置是否正确
2. 收件人邮箱地址是否有效
3. 邮箱是否开启了 SMTP 服务
4. 系统是否启用自动推送能力

### Q5: 自动初筛未触发？

检查项：
1. 岗位是否勾选"上传简历后自动进入初筛"
2. 岗位是否绑定了初筛 Skill
3. 简历上传时是否关联了该岗位

### Q6: 看不到某个功能菜单？

可能原因：
1. 当前角色未分配相应权限
2. 用户数据范围不包含该组织
3. 联系管理员检查权限配置
