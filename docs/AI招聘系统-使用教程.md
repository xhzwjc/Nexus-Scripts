## 0. 核心招聘流程（主流程概览）

> 本章节概括了从系统初始化到完成一次完整招聘的核心流程。如果你是第一次使用本系统，建议先阅读本章建立全局认识，再按需深入了解各功能模块的详细操作。

系统标准招聘流程包含以下 7 个阶段：

| 步骤 | 阶段 | 执行者 | 关键产出 |
|------|------|--------|----------|
| 1 | 用户与权限配置 | 管理员 | 可用账号与角色 |
| 2 | 岗位创建 | 招聘专员 | 带 Skill 绑定的岗位 |
| 3 | JD 生成 | 招聘专员 | 可发布的岗位描述 |
| 4 | Skill 配置 | 管理员 | 评估规则与面试题模板 |
| 5 | 简历上传 | 招聘专员 | 入库候选人 |
| 6 | AI 初筛 | 系统自动/人工触发 | 评分与建议 |
| 7 | 状态流转与后续 | 招聘专员 | 面试 → Offer → 入职 |

---

### 步骤 1：添加用户与配置权限

在使用系统前，管理员需完成基础配置：

1. **进入权限中心** → 添加用户账号
2. **分配角色**：管理员 / 招聘专员 / 只读用户
3. **设置数据范围**：仅本人 / 本部门 / 主组织及下级 / 整个公司 / 集团全部

> 详细操作见 [1. 登录与权限说明](#1-登录与权限说明)

---

### 步骤 2：创建岗位

招聘专员进入 **岗位管理** 页面，点击 **新建岗位**：

1. 填写岗位基本信息（名称、部门、地点、招聘人数等）
2. 在 **Skill 与自动化配置** 区域绑定：
   - **JD Skill**：用于 AI 生成岗位描述
   - **初筛 Skill**：用于 AI 简历初筛评估
   - **面试题 Skill**：用于生成面试题目
3. 配置自动化选项：
   - 勾选 **上传简历后自动进入初筛**（需已绑定初筛 Skill）
   - 勾选 **初筛通过后自动推进候选人状态**
   - 配置 **自动推送邮件**（可选）

> 详细操作见 [4. 岗位管理](#4-岗位管理)

---

### 步骤 3：生成 JD

在岗位工作区的 **当前 JD** 标签页：

1. 在 **AI 生成附加要求** 中填写额外要求（可选）
2. 点击 **AI 生成 JD** 按钮
3. 等待生成完成（状态：排队中 → 生成中 → 已完成）
4. 查看三种视图模式：
   - **可直接发布版**：纯文本，可直接复制到招聘平台
   - **Markdown 源文本**：可编辑格式
   - **预览版**：HTML 渲染效果
5. 点击 **保存新版本** 将 JD 保存到历史版本库

> 详细操作见 [4.2.1 JD 标签页](#421-jd-标签页)

---

### 步骤 4：配置 Skill（前置准备）

管理员进入 **管理设置 → Skill 管理**，为招聘流程准备评估规则：

| Skill 类型 | 用途 | 配置要点 |
|------------|------|----------|
| JD Skill | 生成岗位描述 | 定义角色背景、关键要求、加分项 |
| 初筛 Skill | 简历自动评分 | 设置评估维度、评分规则、录用建议逻辑 |
| 面试题 Skill | 生成面试题 | 设定题目模块、考察方向、难度要求 |

1. 点击 **新增 Skill**
2. 选择 Skill 类型
3. 填写名称、描述、标签
4. 在结构化编辑器中配置具体内容（维度、规则、要求）
5. 保存并启用

> 详细操作见 [12.4 技能库（Skill 管理）](#124-技能库skill-管理)

---

### 步骤 5：上传简历

通过顶部操作栏的 **上传简历** 按钮打开弹窗：

1. **关联岗位**：下拉选择目标岗位（推荐），或选择"暂不关联岗位"
2. **目标组织**：仅在"暂不关联岗位"且多组织时显示
3. **所在城市**：手动指定或自动识别
4. **选择文件**：支持 PDF / DOCX，可批量上传
5. 点击 **开始上传**

上传完成后系统自动解析简历并创建候选人记录，若岗位开启了自动初筛，系统会自动排队处理。

> 详细操作见 [5. 简历上传](#5-简历上传)

---

### 步骤 6：AI 初筛

初筛有三种触发方式：

| 方式 | 触发条件 | 适用场景 |
|------|----------|----------|
| 自动触发 | 岗位开启"上传自动初筛"，且简历关联到该岗位 | 日常批量处理（推荐） |
| 手动单个 | 在候选人详情中点击 **开始初筛** | 针对特定候选人补充评估 |
| 批量触发 | 在候选人列表勾选多人，点击 **批量入队** | 批量补充初筛 |

初筛完成后，候选人状态自动变化：

```
待初筛 → 初筛中 → 初筛通过 / 初筛淘汰 / 初筛失败
```

在候选人详情 **AI 评估** 标签页查看结果：综合评分、维度评分、优势与风险、AI 建议。

> 详细操作见 [6. AI 初筛](#6-ai-初筛)

---

### 步骤 7：状态流转与后续操作

初筛通过后，进入招聘后续流程。根据业务需要，可执行以下操作：

**状态推进**
- 在候选人详情面板底部，点击目标状态按钮
- 可填写变更原因，系统记录完整流转历史

**面试安排**
- 点击 **添加面试安排**，填写轮次、面试官、时间、地点
- 系统按时间倒序展示所有面试安排

**面试题生成**
- 进入 **面试准备** 标签页
- 配置轮次名称和补充要求
- 点击 **面试题** 按钮，AI 自动生成面试题
- 支持流式预览，生成完成后可下载 HTML

**邮件发送**
- 点击 **发送简历**，选择发件箱、收件人、填写主题正文
- 支持单个发送和批量发送
- 若岗位配置了自动推送，初筛完成后符合条件的状态会自动触发邮件

**Offer 管理**
- 点击 **创建 Offer**，填写薪资、部门、入职日期、Offer 内容
- Offer 支持状态流转：草稿 → 已发送 → 已接受 / 已拒绝 / 已撤回

**跟进记录**
- 点击 **添加跟进**，选择类型（备注 / 电话 / 邮件 / 面试 / 其他）
- 填写跟进内容，系统按时间线展示

> 各功能详细操作见对应章节：[7. 候选人管理](#7-候选人管理)、[8. 邮件发送](#8-邮件发送)、[9. 面试题生成](#9-面试题生成)

---

### 流程总结

```
添加用户 → 创建岗位 → 生成 JD → 配置 Skill → 上传简历 → AI 初筛 → 状态流转与后续
   ↑_________________________________________________________|
   └───────────────── 持续迭代，滚动招聘 ──────────────────────┘
```

完成一次招聘流程后，可重复执行步骤 2-7 进行新的招聘。Skill 和模型配置只需首次设置，后续复用即可。

## 1. 登录与权限说明

### 1.1 登录

1. 在浏览器中打开系统地址
2. 在 **Key** 输入框中输入你的访问密钥
3. 点击 **验证密钥** 按钮
4. 验证通过后自动进入系统首页

> ⚠️ 注意：密钥仅在创建时显示一次。如遗忘请联系管理员重置。

[截图：登录页面]

### 1.2 权限体系

系统采用基于角色的权限控制（RBAC）。不同角色拥有不同的操作权限：

| 角色类型 | 可访问内容 |
|----------|-----------|
| 管理员 | 全部功能，包括 Skill 管理、模型配置、邮件中心、权限中心 |
| 招聘专员 | 工作台、岗位管理、候选人管理、初筛、面试题、邮件发送 |
| 只读用户 | 仅查看工作台、候选人列表、审计日志 |

> 如果你看不到某个功能入口（如顶部的"管理设置"），说明当前账号未被授予管理权限，请联系管理员。

### 1.3 组织与数据范围

每个用户绑定一个主组织，并通过"数据范围"控制可见数据：

| 数据范围 | 说明 |
|----------|------|
| 仅本人 | 只能查看自己创建的数据 |
| 本部门 | 可查看本部门及子部门数据 |
| 主组织及下级 | 可查看主组织及所有下级组织数据 |
| 整个公司 | 可查看公司下所有数据 |
| 集团全部 | 可查看集团下所有数据 |

## 3. 工作台

工作台是进入系统后的默认页面，提供招聘全局总览。

[截图：工作台页面]

### 3.1 今日总览横幅

页面顶部显示"招聘推进总览"卡片，包含三个核心数字：招聘中岗位数、待初筛候选人、今日新增简历。右侧有"打开 AI 招聘助手"和"新建岗位"两个快捷按钮。

### 3.2 指标卡片

八个指标卡片按 4 列网格排列：

| 指标 | 说明 |
|------|------|
| 招聘中岗位 | 状态为"招聘中"的岗位数量 |
| 今日新增简历 | 今天导入的候选人数量 |
| 待初筛候选人 | 等待 AI 初筛的候选人数量 |
| 今日初筛通过 | 今天通过初筛的候选人数量 |
| 今日 AI 处理数 | 今天触发的 AI 任务总数 |
| 待发布岗位 | 草稿状态的岗位数量 |
| 待安排面试 | 初筛通过但尚未安排面试的候选人数量 |
| 待确认结果 | 等待最终录用决定的候选人数量 |

### 3.3 快捷操作

六个快捷操作卡片，点击直接执行对应操作：

- **新建岗位** → 打开岗位创建弹窗
- **生成 JD** → 跳转到岗位管理页面
- **上传简历** → 打开简历上传弹窗
- **批量初筛** → 跳转到候选人页面
- **生成面试题** → 跳转到候选人页面
- **打开 AI 助手** → 打开 AI 助手抽屉

### 3.4 底部面板

页面底部四个面板组成 2 列网格：

1. **最近候选人**：最多显示 8 位最近更新的候选人，点击任意一行跳转到候选人页面并自动选中该候选人
2. **AI 助手**：嵌入式 AI 对话工作区，可直接在工作台与 AI 交互
3. **最近 AI 任务**：最多显示 6 条最近的 AI 任务日志，点击跳转到审计中心
4. **状态分布**：按候选人状态分组统计数量，帮助识别招聘漏斗瓶颈

### 3.5 招聘漏斗

工作台新增**招聘漏斗**可视化面板，展示从简历导入到最终入职的全流程转化数据：

| 环节 | 说明 |
|------|------|
| 新导入 | 刚进入系统的候选人数量 |
| 待初筛 | 尚未完成 AI 初筛的候选人 |
| 初筛通过 | 评分达标进入后续流程 |
| 面试通过 | 通过面试评估 |
| 已发 Offer | Offer 已发出 |
| 已入职 | 候选人已入职 |
| 已淘汰 | 各环节淘汰合计 |
| 人才库 | 进入人才储备库 |

漏斗图直观展示各环节的转化率，帮助管理层快速识别瓶颈所在。

### 3.6 候选人来源分布

工作台新增**候选人来源分布**统计面板，按候选人来源渠道统计数量：

| 来源 | 说明 |
|------|------|
| 手动上传 | 通过系统上传功能导入 |
| 邮件投递 | 通过邮件渠道投递 |
| 未知来源 | 未识别到明确来源 |

点击来源名称可快速筛选对应来源的候选人。

## 4. 岗位管理

### 4.1 岗位列表

进入岗位管理页面后，左侧显示岗位列表。

**筛选与搜索**：
- 顶部搜索框支持按岗位名称、部门、地点搜索
- 状态筛选芯片：全部 / 草稿 / 招聘中 / 暂停中 / 已关闭
- 列表上方显示可见数量/总数

**列表项信息**：每个岗位卡片显示岗位名称、组织标签（多组织时显示）、状态徽标、候选人数、部门、地点。

**操作**：点击任意岗位卡片，右侧打开岗位工作区。

[截图：岗位列表页面]

### 4.2 岗位工作区

选中岗位后，右侧显示岗位工作区，包含两个标签页：**当前 JD** 和 **岗位配置**。

顶部信息栏显示：岗位名称、招聘人数、JD 版本数、候选人数、最后更新时间。右上角有"版本与关联"按钮，点击展开/收起侧边面板。

#### 4.2.1 JD 标签页

**AI 生成 JD**：
1. 在"AI 生成附加要求"文本框中填写额外要求（可选）
2. 点击 **AI 生成 JD** 按钮
3. 等待生成完成，状态徽标会实时显示：排队中 → 生成中 → 已完成
4. 生成过程中可点击 **停止生成** 按钮取消

> ⚠️ 注意：如果生成失败，错误信息会显示在 JD 区域上方。请检查模型配置是否正确。

**三种 JD 视图模式**：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 可直接发布版 | 纯文本格式，可直接复制到招聘平台 | 发布到 BOSS 直聘、智联招聘等 |
| Markdown 源文本 | 可编辑的 Markdown 格式 | 需要手动调整内容时 |
| 预览版 | HTML 渲染预览 | 查看最终排版效果 |

**版本管理**：
- **一键复制发布文案**：复制当前 JD 的纯文本内容到剪贴板
- **保存新版本**：将当前 JD 保存为新版本（可填写版本标题和备注）
- **保存后设为生效版本**：勾选后保存时自动激活该版本
- **重新生成**：使用当前附加要求重新生成 JD

[截图：JD 标签页三种视图模式]

#### 4.2.2 岗位配置标签页

显示岗位的完整配置信息，包括：

- 基本信息：部门、组织、地点、用工类型、薪资范围、招聘人数、标签
- 关键要求和加分项
- 技能绑定情况：JD Skill、初筛 Skill、面试题 Skill
- 自动化配置：自动初筛、自动推进、自动推送邮件
- 岗位摘要

操作按钮：
- **编辑岗位**：打开岗位编辑弹窗
- **发布岗位**：打开发布任务弹窗
- **删除岗位**：弹出确认对话框后删除

## 6. AI 初筛

### 6.1 什么是 AI 初筛

AI 初筛是系统的核心功能。系统会调用配置好的大模型，根据岗位绑定的初筛 Skill（评估规则），对候选人简历进行自动评分和分析。

初筛结果包括：综合评分（0-100）、各维度评分、优势与风险分析、录用建议。

### 6.2 触发初筛的方式

#### 方式一：自动触发（推荐）

在岗位配置中开启"上传简历后自动进入初筛"后，上传到该岗位的简历会自动触发 AI 初筛，无需手动操作。

前提条件：
- 岗位已绑定至少一个初筛 Skill
- 上传简历时选择了该岗位

#### 方式二：单个候选人手动触发

1. 在候选人列表中点击目标候选人，打开详情面板
2. 在 **AI 评估** 标签页的工具栏中，点击 **开始初筛** 按钮
3. 等待初筛完成，状态会实时更新

#### 方式三：批量触发

1. 在候选人列表中勾选多个候选人（支持逐个勾选、全选、或拖拽框选）
2. 顶部批量操作栏出现后，点击 **批量入队** 按钮
3. 系统为每个候选人创建初筛任务，完成后自动刷新列表

> 💡 提示：如果某个候选人已有进行中的初筛任务，系统会自动跳过并复用现有任务。

#### 批量初筛的批次管理

系统支持对批量初筛进行批次级别的管理和追踪：

**查询批次状态**：
在批量初筛发起后，系统会为该批次分配唯一的 `batch_id`。可在审计中心或相关入口查询该批次的整体执行进度。

**取消批量初筛**：
对于尚未全部完成的批量初筛批次，可发起批次取消操作。系统会尝试取消该批次下所有排队中和运行中的初筛任务。

> ⚠️ 注意：已完成的任务无法取消，取消操作仅影响尚未执行完毕的任务。

### 6.3 初筛状态

初筛过程中，候选人状态会自动变化：

| 状态 | 说明 |
|------|------|
| 待初筛 | 等待初筛开始 |
| 初筛中 | AI 正在评估 |
| 初筛通过 | 评分达标，进入下一阶段 |
| 初筛淘汰 | 评分未达标 |
| 初筛失败 | 系统异常导致失败 |

### 6.4 停止初筛

- **单个**：在候选人详情面板中点击 **停止初筛** 按钮
- **批量**：在批量操作栏中点击 **停止批量初筛** 按钮

### 6.5 查看初筛结果

初筛完成后，在候选人详情的 **AI 评估** 标签页中查看：

- **总评分**：0-100 分
- **匹配度**：与岗位要求的匹配百分比
- **AI 建议**：推荐的后续状态和面试重点
- **优势**：候选人的优势列表
- **风险**：候选人的潜在风险列表
- **维度评分**：各评估维度的分数和依据
- **评分验证警告**：当 AI 输出结构与预期存在偏差时，系统会以橙色徽标提示，供人工复核参考

**筛选模式**：
系统在初筛时支持多种执行模式，满足不同场景需求：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 默认模式 | 标准初筛流程，解析+评分分步执行 | 日常初筛 |
| 一体化模式 | 简历解析与评分合并为一次调用 | 需要快速处理的场景 |
| 仅评分模式 | 复用已解析结果，只执行评分 | 重新评估候选人 |

点击 **查看完整 AI 输出** 按钮，可查看 AI 的完整评估报告。弹窗中支持：
- **复制全文**：一键复制所有内容
- **查看结构化原始输出**：展开查看 JSON 格式的原始数据

## 7. 候选人管理

> 💡 **系统实时推送**：系统采用 SSE（Server-Sent Events）长连接技术，前端建立一次连接后，后端会主动推送任务状态变化、候选人更新等事件。这意味着列表数据、候选人状态、AI 任务进度等都会自动刷新，无需手动点击刷新按钮。

### 7.1 候选人列表

候选人页面是招聘工作的核心操作界面。

[截图：候选人列表页面]

#### 7.1.1 视图模式

支持两种视图模式，通过列表上方的按钮切换：

| 模式 | 说明 |
|------|------|
| 列表视图 | 表格形式，支持列宽调整、水平滚动 |
| 看板视图 | 按状态分列的看板形式，直观展示招聘漏斗 |

#### 7.1.2 筛选栏

点击 **筛选** 按钮展开/收起筛选栏。支持六种筛选条件：

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 搜索 | 文本输入 | 按姓名、手机号、邮箱、公司搜索 |
| 岗位 | 多选下拉 | 按一个或多个岗位筛选 |
| 状态 | 多选下拉 | 按一个或多个候选人状态筛选 |
| 匹配度 | 单选下拉 | 全部 / 80% 以上 / 60% 以上 / 40% 以上 |
| 来源 | 多选下拉 | 按候选人来源筛选 |
| 时间 | 单选下拉 | 全部时间 / 今天 / 近 7 天 / 近 30 天 |

筛选栏下方显示当前筛选结果数量："共匹配到 N 位候选人"。点击 **重置** 按钮清空所有筛选条件。

#### 7.1.3 列表视图

列表包含以下可调整列宽的列：

| 列名 | 内容 |
|------|------|
| 复选框 | 用于选择候选人以执行批量操作 |
| 候选人 | 姓名、年龄、手机号/邮箱、已发简历徽标 |
| 组织 | 所属组织（多组织时显示） |
| 岗位 | 关联岗位名称，未关联时显示"未分配岗位" |

| 匹配度 | 匹配百分比 |
| 城市 | 候选人所在城市 |
| 来源 | 候选人来源（如手动上传、邮件投递等） |
| 更新时间 | 最后更新时间 |

**列宽调整**：拖动列头边框可调整列宽，双击列头边框恢复默认宽度。

**水平滚动**：当列数超出可视区域时，底部出现水平滚动条。也可按住 Shift + 滚轮进行水平滚动。

#### 7.1.4 看板视图

看板视图按候选人状态分列显示，每列顶部显示状态名称和数量。每个候选人卡片显示姓名、岗位、匹配度、更新时间。点击卡片选中候选人。

#### 7.1.5 选择候选人

支持三种选择方式：

| 方式 | 操作 |
|------|------|
| 单选 | 点击候选人行左侧的复选框 |
| 全选 | 点击表头的全选复选框 |
| 拖拽框选 | 在列表空白区域按住鼠标拖拽，画出矩形框选区域，所有与矩形相交的行会被选中。按住 Shift 拖拽可追加到已有选择中 |

### 7.2 批量操作

选中候选人后，列表上方出现批量操作栏，显示"已选中 N 位候选人"。包含以下操作：

| 按钮 | 说明 |
|------|------|
| 清空选择 | 取消所有选中状态 |
| 批量入队 | 为选中候选人批量触发 AI 初筛 |
| 停止批量初筛 | 取消所有正在运行的批量初筛任务（仅在初筛运行中显示） |
| 批量发送简历 | 打开简历邮件弹窗，收件人包含所有选中候选人 |

| 导出 | 导出选中候选人数据（含或不含简历附件） |
| 批量删除 | 删除所有选中候选人（需确认，有进行中 AI 任务的候选人会被跳过） |

- 匹配度百分比
- 评分验证警告（如有，以橙色徽标显示）

**优势与风险**：编号列表展示

**维度评分**：每个维度显示分数、满分、评估依据列表

管理简历文件和生成面试题。

- Skill 选择器：点击芯片切换选择。选中的芯片高亮显示。点击"恢复默认 Skills"回到系统自动选择

**Skill 来源优先级**：岗位绑定的 Skill > 面试工作流记忆 > 无 Skill

**面试题卡片**（生成后显示）：

## 10. AI 招聘助手

AI 招聘助手是一个基于大模型的对话式工作区，可以执行招聘相关的各种操作。

### 10.1 打开方式

| 方式 | 说明 |
|------|------|
| 顶部操作栏 | 点击 **打开 AI 助手** 按钮，以抽屉模式打开 |
| 工作台 | 使用嵌入式 AI 助手面板，或点击"打开完整助手" |
| 候选人详情 | AI 评估标签页中的 **打开 AI 助手** 按钮，自动关联当前候选人 |
| 左侧导航 | 点击"招聘助手"进入独立页面模式 |
| 浮动按钮 | 页面右下角的浮动 AI 按钮，悬停展开后点击 |

### 10.2 四种显示模式

| 模式 | 说明 | 切换方式 |
|------|------|----------|
| 页内模式 | 嵌入在助手页面中 | 助手头部的"页内"按钮 |
| 抽屉模式 | 从右侧滑出的面板（默认，最宽 1360px） | 助手头部的"浮层"按钮 |
| 全屏模式 | 覆盖整个屏幕 | 助手头部的"全屏"按钮 |
| 工作台模式 | 嵌入在工作台页面中的紧凑版 | 工作台页面的 AI 助手面板 |

### 10.3 上下文面板

点击助手头部的 **上下文** 按钮展开/收起上下文面板，包含：

- **当前岗位**：下拉选择，AI 会基于该岗位的 JD 和配置回答问题
- **激活 Skills**：点击芯片选择/取消，AI 回答时会参考这些 Skill
- **当前模型**：下拉切换对话使用的 AI 模型

### 10.4 对话功能

**发送消息**：在输入框中输入问题，按 **Ctrl/Cmd + Enter** 发送，或点击发送按钮。

**预设提示词**：输入框上方有多个可点击的提示词芯片，点击直接发送预设问题：
- 为当前岗位生成 JD
- 查看当前岗位的候选人
- 为当前候选人生成面试题
- 重新初筛当前候选人并追加硬性条件
- 说明这次对话使用了哪些 Skills
- 当前使用的是什么模型
- 更多 / 收起

**AI 回复**：
- 支持流式输出，实时显示生成过程
- 显示使用的模型信息和 Skill 来源
- 当 AI 回退到确定性结果时会显示回退标记
- 点击 **回到底部** 按钮快速跳转到最新消息

**澄清请求**：当 AI 需要更多信息时，会显示多个选项按钮供用户选择。

**邮件确认**：当 AI 准备发送邮件时，会显示邮件预览卡片，包含收件人、主题、正文等信息。可选择：
- **确认发送**：直接发送
- **编辑后发送**：打开完整邮件编辑弹窗
- **再次发送**：发送成功后可重新发送

**流式输出**：AI 回复采用 SSE 流式传输，内容实时逐字呈现，无需等待全部生成完毕即可开始阅读。

**停止生成**：在 AI 生成过程中，点击 **停止** 按钮可中断生成。

### 10.5 上下文记忆

系统会自动保存 AI 助手的对话上下文，包括：

- **当前选中的岗位**：AI 会基于该岗位的 JD 和配置回答问题
- **当前候选人**：从候选人详情打开助手时自动关联
- **激活的 Skills**：用户手动选择或系统默认的 Skill 集合

上下文记忆会在每次对话后自动保存，下次打开助手时自动恢复，确保对话连续性。

### 10.6 JD 流式生成

在 AI 助手中可直接发起 JD 的流式生成：

1. 向 AI 助手发送"为当前岗位生成 JD"等指令
2. AI 助手会实时流式输出 JD 内容
3. 生成过程中可随时停止
4. 生成完成后可选择直接保存到当前岗位

> 💡 提示：与岗位工作区的 JD 生成相比，助手中的流式生成更适合边生成边与 AI 交互调整需求的场景。

### 12.5 资源治理与共享

> 入口：管理设置 → 资源治理

资源治理模块统一管理 Skill、AI 模型、发件箱、收件人四类资源的共享策略，实现跨组织、跨团队的安全协作。

#### 资源共享策略

每项资源（Skill、模型、发件箱、收件人）都包含以下治理字段：

| 字段 | 说明 | 影响范围 |
|------|------|----------|
| 所属组织 | 资源归属的组织代码 | 决定资源在哪个组织下创建和管理 |
| 作用域级别 | 资源的可见范围 | 如 ORG 级别表示同组织可见 |
| 共享策略 | PRIVATE / SHARED 等 | PRIVATE 仅创建者可见；SHARED 可被授权用户查看和使用 |
| 允许子组织使用 | true / false | 下级组织是否能使用上级组织的资源 |
| 允许复制 | true / false | 其他用户是否能复制该资源到自己的组织 |

#### 资源复制

对于允许复制的资源，任何有权限的用户都可以：

1. 在资源治理页面或对应管理页面找到目标资源
2. 点击 **复制** 按钮
3. 选择目标组织（可选，默认同组织）
4. 系统创建该资源的副本，配置键自动加 `-copy` 后缀
5. 复制后的资源归当前用户所有，可独立编辑

> 💡 提示：资源复制功能特别适用于以下场景：
> - 总部创建标准 Skill / 模型模板，各分公司复制后微调使用
> - 团队协作时快速基于优秀配置创建新资源
> - 跨组织迁移配置时避免重复录入

## 13. 候选人状态流转说明

系统定义了 13 个候选人状态，覆盖从导入到入职的完整流程：

| 状态 | 中文名 | 说明 |
|------|--------|------|
| `new_imported` | 新导入 | 刚导入系统，尚未进入任何流程 |
| `pending_screening` | 待初筛 | 等待 AI 初筛 |
| `screening_running` | 初筛中 | AI 正在评估 |
| `screening_failed` | 初筛失败 | 系统异常导致失败 |
| `screening_passed` | 初筛通过 | 评分达标 |
| `screening_rejected` | 初筛淘汰 | 评分未达标 |
| `pending_interview` | 待面试 | 等待安排面试 |
| `interview_passed` | 面试通过 | 面试评估通过 |
| `interview_rejected` | 面试淘汰 | 面试评估未通过 |
| `pending_offer` | 待 Offer | 等待发出 Offer |
| `offer_sent` | 已发 Offer | Offer 已发出 |
| `hired` | 已入职 | 候选人已入职 |
| `talent_pool` | 人才库 | 进入人才储备库 |

**状态变更方式**：
1. **手动变更**：在候选人详情的档案标签页中点击状态按钮
2. **自动变更**：开启"初筛通过后自动推进状态"后，初筛通过的候选人自动从"待初筛"变为"初筛通过"
3. **AI 建议**：AI 初筛结果中会推荐候选人的后续状态

## 14. 常见问题

### Q1: 登录失败，密钥无效？

- 检查密钥是否输入正确（区分大小写）
- 确认密钥未过期
- 联系管理员重置密钥

### Q2: 无法发起初筛？

可能原因：
1. **候选人未关联岗位** → 在候选人详情的档案标签页中选择岗位
2. **岗位未绑定初筛 Skill** → 编辑岗位，添加初筛 Skill
3. **候选人无简历** → 先上传简历

### Q3: 自动初筛未触发？

检查项：
1. 岗位是否勾选了"上传简历后自动进入初筛"
2. 岗位是否绑定了初筛 Skill
3. 上传简历时是否选择了该岗位

### Q4: 无法生成面试题？

可能原因：
1. **候选人未关联岗位** → 先为候选人分配岗位
2. **模型配置不正确** → 检查面试题生成任务类型的模型配置

### Q5: 邮件发送失败？

检查项：
1. 发件箱 SMTP 配置是否正确
2. 收件人邮箱地址是否有效
3. 邮箱是否开启了 SMTP 服务
4. 使用邮件中心页面的发件箱编辑功能测试连接

### Q6: 看不到某个功能菜单？

可能原因：
1. 当前账号未被授予相应管理权限
2. 用户数据范围不包含该组织
3. 联系管理员检查权限和数据范围配置

### Q7: 列表数据不更新？

- 点击顶部操作栏的 **刷新** 按钮手动刷新
- 检查组织切换器是否选对了组织
- 检查筛选条件是否过严

### Q8: 批量删除候选人时部分被跳过？

- 有正在进行中的 AI 任务的候选人无法删除
- 需先等待或取消该候选人的 AI 任务，再执行删除

### Q9: 如何管理候选人的面试安排？

在候选人详情面板中向下滚动，找到"面试安排"区域：
1. 点击 **添加面试安排** 按钮
2. 填写面试轮次、面试官、时间、地点等信息
3. 提交后系统会自动保存并与候选人的岗位关联
4. 面试安排按时间排序，支持删除操作

### Q10: 如何创建和管理 Offer？

在候选人详情面板中向下滚动，找到"Offer 管理"区域：
1. 点击 **创建 Offer** 按钮
2. 填写 Offer 标题、薪资、部门、入职日期、Offer 内容等信息
3. 创建后 Offer 默认为"草稿"状态
4. 点击状态下拉框可将其变更为"已发送""已接受""已拒绝"或"已撤回"

### Q11: 为什么其他用户看不到我创建的 Skill / 模型 / 发件箱？

请检查该资源的**共享策略**和**作用域级别**：
- 若共享策略为 PRIVATE，仅创建者本人可见
- 若共享策略为 SHARED 但作用域不匹配，对方可能不在可见范围内
- 若需要子组织使用，需勾选"允许子组织使用"
- 若希望其他用户基于您的配置创建自己的版本，需勾选"允许复制"

管理员可在**资源治理**页面统一调整这些策略。

### Q12: AI 生成的 Skill 可以直接使用吗？

AI 生成的 Skill 建议作为**参考模板**使用：
1. AI 生成后，先查看内容是否符合实际评估需求
2. 根据需要调整评分维度、权重、Prompt 措辞等细节
3. 保存为正式 Skill 并启用后，建议先用少量候选人测试效果
4. 确认效果满意后再大规模投入生产使用
