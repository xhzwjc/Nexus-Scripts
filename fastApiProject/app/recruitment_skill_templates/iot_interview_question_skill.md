# IoT 测试工程师面试题 Skill

## 岗位 JD（固定，用户无需重复发送）

**岗位名称**：IoT测试工程师 / 软硬件测试工程师

**岗位职责**：负责智能家居IoT产品全流程测试（智能家电、门锁、灯具、网关等及配套APP/小程序）；参与需求评审、制定测试计划方案、设计测试用例（手动+自动化）；搭建维护IoT测试环境（星闪、Wi-Fi、蓝牙、ZigBee、Thread等协议、跨品牌联动）；执行测试、定位缺陷、跟踪修复；输出测试报告/缺陷报告/用例文档；关注行业动态，优化测试流程；配合研发产品团队解决技术问题。

**任职要求**：本科及以上，计算机相关专业（电子信息、自动化、软件工程等），2-3年智能家居/IoT测试经验；熟悉星闪、Wi-Fi(802.11)、BLE、ZigBee、Thread、MQTT等协议测试要点；了解米家、华为鸿蒙、苹果HomeKit互联互通逻辑；掌握软件测试基础，具备独立设计用例、性能/功能测试、缺陷定位能力；熟悉至少一种自动化测试工具（Appium/Espresso/UIAutomator/Python+Selenium/Requests）。

---

## 全程铁律（任何情况下不得违反）

1. **严禁虚构**：所有题目必须基于候选人简历原文、岗位 JD 原文和当前 Skill 规则生成
2. **强制出题**：无论初筛结论如何，只要用户要求出面试题，必须出题
3. **软件测试不为零**：面试题中软件测试模块至少 1 题
4. **AI工具必出**：每份面试题必须包含 1 道 AI 工具使用考察题
5. **规则变更原则**：用户告知规则变动时，只改动指定点，其他所有规则保持不变

## 模块二：面试题生成

**触发方式**：初筛完成后，用户说"给我XX的面试题"，即触发本模块。

**强制规则**：无论初筛是否通过，只要用户要求出题，必须直接出题，不得拒绝。初筛未通过者，在Header区域明确标注「初筛未达标·强制面试」并注明重点考察方向。

### 每道题必须包含的五个部分

1. **面试题正题**：针对候选人简历和JD定制，不得虚构内容，所有技术点必须真实
2. **参考答案要点**：列出关键评判点，区分不同情况（如候选人声称用过 vs 坦承未用过）
3. **通过/注意/一票否决判定**：明确告诉面试官什么样的回答算通过、什么算注意、什么直接一票否决终止面试
4. **追问环节**：候选人答出后使用，深挖细节，1-2条，体现面试官专业性
5. **答不出时·面试官解释**：面试官参考此内容进行解释，说明为什么要问这道题

### 模块时长分配（总计约 40 分钟）

| 模块 | 建议时长 | 要求 |
|------|----------|------|
| 智能家居生态模块（重点） | 约12分钟 | 米家/鸿蒙/HomeKit/涂鸦等真实联动测试经历必问 |
| IoT通信协议模块 | 约10分钟 | BLE配网、ZigBee组网稳定性、MQTT消息可靠性等 |
| 软件测试基础模块 | 约8分钟 | 至少1题，不可为0 |
| 硬件能力模块 | 约5分钟 | 重点是设备上手测试、问题复现和系统联调 |
| 嵌入式/固件测试模块 | 约3分钟 | 仅做基本确认，了解OTA概念即可 |
| AI工具使用模块 | 约2分钟 | 必出1题，考察是否用过AI及如何融入测试工作 |

### HTML 输出规范

- 输出完整可用的单文件 HTML，所有 CSS 和 JS 内联在同一文件中
- 左栏展示题目主体，右栏展示专业词释义
- 顶部 Header 必须包含候选人姓名、工作年限、学历专业、初筛状态、重点考察方向、总时长
- 输出不得退化成泛化题库；每个模块都必须明显对应当前 Skill

### 专业词释义规范

- 释义必须来源于权威文档（IEEE/IEC/ISO/OASIS/厂商官方文档等）
- 每条注明来源
- 严禁自造定义
